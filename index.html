Projet 1 : Pepe Cleaner

Pepe cleaner est un jeu vidéo réalisé par une équipe de 6 personnes dans le cadre d'un hakathon. Dans ce groupe, nous étions 3 développeurs et 3 designers. Ce projet avait pour but de réalisé un jeu vidéo dans un temps imparti de 48h tout en respectant un thème imposé. Pour ce jeu, nous avons donc utilisé godot engine, un moteur de jeu open source. Ce projet nous a permis de mettre en pratique nos compétences en développement de jeu vidéo mais également de travailler en équipe avec des profils totalement différents.

Projet 2 : Puzzles & Ridles

Ce projet a été réalisé dans le cadre de notre formation à l'IUT d'Arles. Il s'agit d'une application mobile et web permettant de jouer à des jeux puzzle comme le sudoku, les môts croisés, etc. Pour effectuer cette application, j'ai organisé mon projet en deux parties : une partie Back-end commune à l'application mobile et web et une partie Front-end spécifique à chaque support. Pour la partie back-end j'ai choisi d'utiliser Python avec le framework Flask pour mettre en place une API Rest. Pour la partie front-end, j'ai utilisé React pour l'aspect web et Ionic pour la partie mobile. Ce projet a été continuellement mis à jour et amélioré tout au long de ma 3ème année d'IUT. Ce fût un réel défi de construire tout une architecture logiciel de A à Z.

Projet 3 : Intégration API Tele2 - CFast

Ce projet a été mené au sein de l'entreprise Telcosud, spécialisée dans la gestion de cartes SIM professionnelles pour l'IoT/M2M. L'objectif était de synchroniser automatiquement les données entre deux logiciels métiers critiques : Tele2 (gestion de cartes SIM) et CFast (facturation client).

J'ai étudié et modélisé les APIs REST de ces deux plateformes, en m'appuyant sur des schémas BPMN pour documenter les flux, contraintes, et scénarios d'erreur. J'ai également produit un plan d'action opérationnel listant les appels nécessaires, les dépendances entre les entités (contrats, clients, abonnements, etc.) et les prérequis de sécurisation.

Ce projet a développé chez moi une posture d'intégrateur : capacité à analyser des systèmes hétérogènes, à anticiper les erreurs, et à proposer des solutions fiables et maintenables. Une documentation complète a été rédigée afin d'assurer la pérennité de l'outil.
